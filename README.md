# Nexus Blade — итоговый TypeScript-проект под Яндекс Игры

Это готовый каркас игры на **React + TypeScript + Vite + Tailwind** с интеграцией **Yandex Games SDK**.

## Что внутри

- `index.html` — входная HTML-страница и подключение SDK.
- `src/main.tsx` — точка входа React.
- `src/App.tsx` — основная игровая логика.
- `src/styles.css` — глобальные стили, отключение скролла, safe mobile behavior.
- `src/lib/yandexGames.ts` — обёртка над SDK Яндекс Игр.
- `src/types/yandex-games.d.ts` — локальные типы для глобального `window.YaGames`.
- `package.json` — все зависимости и npm-скрипты.
- `vite.config.ts` — конфиг Vite.
- `tailwind.config.js`, `postcss.config.js` — конфиги Tailwind/PostCSS.
- `tsconfig*.json` — конфиги TypeScript.

---

## 1. Что установить

Сначала установи **Node.js LTS**. Проверка:

```bash
node -v
npm -v
```

Потом зайди в папку проекта и установи зависимости:

```bash
npm install
```

В проект уже включён пакет типизации SDK:

```bash
npm install --save-dev @types/ysdk
```

Если ты уже выполнил обычный `npm install`, отдельно это повторять не нужно — пакет уже есть в `package.json`.

---

## 2. Как запустить локально

Из корня проекта:

```bash
npm run dev
```

После этого Vite обычно поднимет сервер примерно на:

```bash
http://localhost:5173
```

Если хочешь открыть с телефона в одной сети, Vite уже настроен с `host: true`.

## 3. Как собрать production build

```bash

```

Готовая сборка появится в папке:

```bash
dist
```

## 4. Как подключить SDK Яндекс Игр

### Для локальной разработки

В `index.html` уже стоит этот вариант:

```html
<script src="https://sdk.games.s3.yandex.net/sdk.js"></script>
```

Он подходит, когда ты запускаешь игру локально или на своём домене.

### Для публикации архивом в Яндекс Игры

Перед загрузкой архива в консоль Яндекс Игр замени строку в `index.html` на:

```html
<script src="/sdk.js"></script>
```

То есть правило простое:

- локальная разработка / свой домен → абсолютный URL;
- архив для консоли Яндекс Игр → относительный `/sdk.js`.

## 5. Порядок запуска без боли

### Вариант A — просто посмотреть локально

1. Распакуй проект.
2. Открой терминал в папке проекта.
3. Выполни:

```bash
npm install
npm run dev
```

4. Открой локальный адрес из терминала.

### Вариант B — подготовить архив для Яндекс Игр

1. Открой `index.html`.
2. Замени:

```html
<script src="https://sdk.games.s3.yandex.net/sdk.js"></script>
```

на:

```html
<script src="/sdk.js"></script>
```

3. Собери проект:

```bash
npm run build
```

4. Содержимое папки `dist` загрузи в архив для публикации.

## 7. На что обратить внимание перед публикацией

### Обязательно проверь:

- что звук реально прекращается при сворачивании вкладки;
- что нет вертикального скролла страницы;
- что кнопки не обрезаются на мобильных экранах;
- что после рекламы игра корректно возвращается в нужное состояние;
- что leaderboard name в коде совпадает с тем, что ты создал в консоли Яндекс Игр.

Сейчас в коде используется:

```ts
const LEADERBOARD_NAME = 'nexus_blade_score';
```

Если в консоли у тебя другое имя, замени его в `src/App.tsx`.

## 9. Если что-то не запускается

### Ошибка `npm is not recognized`
Не установлен Node.js.

### Ошибка `YaGames is not defined`
SDK не подключён или вызывается раньше, чем загрузился script.

### Не работают стили
Проверь, что зависимости Tailwind/PostCSS установились без ошибок через `npm install`.

### Белый экран
Открой DevTools → Console и посмотри первую красную ошибку. Обычно это импорт, путь или опечатка.
